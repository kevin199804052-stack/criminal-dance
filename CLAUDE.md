## 溝通與工作方式
- **一律繁體中文**，白話、分步、少術語（使用者程式背景很淺）。
- **改完一定先在預覽實測「真的解決了使用者提出的問題」再回報**，不要只看程式碼就說修好。
- 每次改動把右下角 **build 版本號 +1**，並提醒使用者 **Ctrl+F5** 確認載到新版。

## 技術與架構
- **純 HTML + JavaScript + CSS，全部在單一檔，無編譯／打包**，瀏覽器直接開。
- 外部相依用 CDN `<script>`（線上版用 Firebase compat SDK）。
- 本機**沒有 node / python**。預覽用 PowerShell 靜態伺服器：`.claude/launch.json` 的 `static` 設定 → `.claude/serve.ps1`（port 8123），用 Claude Preview 的 `preview_start` 啟動。
- **架構原則：引擎（純邏輯、不碰 DOM）與 UI（呈現／互動）分離**，目的是可測試、且本機與線上共用同一套規則引擎。線上版靠「只發給每位玩家自己的視圖」來保護隱藏手牌。
  - 現況：目前 `criminal-dance.html` 仍是邏輯與 UI 混在一起的單體；之後做線上本體時要朝引擎/UI 分離重構。

## 遊戲規格
- **標準版、固定 4 人**。每人發 4 張、共用 **16 張**（必含 第一發現者×1、犯人×1、偵探×1、不在場證明×1、共犯×1，其餘 11 張任選）。
- 持「第一發現者」者開場（必須先打出它），之後輪流，每回合打 1 張並結算。

### 卡表（標準版 12 種 / 32 張；4 人局取 16 張）
| 卡片 | 張數 | 效果 |
|------|------|------|
| 第一發現者 | 1 | 開場必打，無其他效果 |
| 犯人 | 1 | 只有手牌剩 1 張時可打，打出即逃脫獲勝 |
| 偵探 | 4 | 手牌≤3可打；指認某人是犯人，中即勝（對方有不在場證明可免疫） |
| 不在場證明 | 5 | 與犯人同持時，被指認可宣告「不是犯人」（**不需棄掉、可重複**）；單獨打出無效果 |
| 共犯 | 2 | 加入犯人陣營，與犯人同勝負 |
| 目擊者 | 3 | 偷看某人全部手牌（含效果說明） |
| 情報交換 | 3 | 全體同時各傳一張給左邊；**每人自己選**要傳哪張（非隨機） |
| 謠言 | 4 | 全體同時從右邊盲抽一張 |
| 交易 | 5 | 與指定一人**雙方各自選一張**互相盲換 |
| 少年 | 1 | 得知誰持有犯人牌 |
| 神犬（いぬ） | 1 | 指定一人**由他自己選一張棄掉**；棄到犯人牌→神犬勝，否則神犬牌交給他 |
| 一般人 | 2 | 無效果 |

### 計分與賽末
- 偵探破案：偵探 +2、其他平民 +1、犯人陣營 0。
- 神犬立功：神犬 +3、其他平民 +1、犯人陣營 0。
- 犯人逃脫：犯人與所有共犯各 +2、其他人 0。兼具犯人＋共犯者只 +2 不重複。
- **賽末門檻**：常數 `WIN_SCORE`（預設 10）。跨局累積先達門檻**且唯一最高者**贏整場；同分並列續賽。
- `nextRound()` 保留總分重新發牌；`resetAll()` 歸零回設定。

## 兩種模式
- **本機同桌**（完成）：共用一台裝置輪流，靠「換人遮擋畫面」保護手牌。
- **線上連線**（進行中）：各自裝置、用房號加入。後端 Firebase Realtime Database。

## Firebase（線上版）
- 設定 `firebaseConfig` 寫在 `criminal-dance.html` 與 `fb-test.html` 內，專案 `criminal-dance`，DB 在 `asia-southeast1`。
- apiKey 放前端是正常安全的；**資料庫目前是「測試模式」（任何人可讀寫），正式開放遊玩前必須設安全規則**。
- 連線資料模型（大廳）：`rooms/{房號}/ { host, status, players/{playerId}:{name}, createdAt }`。

## 目前進度（持續更新）— build v37（已上 GitHub Pages：github.com/kevin199804052-stack/criminal-dance）
- ✅ 本機同桌完整可玩：12 種牌、出牌限制、三種勝利、計分、賽末門檻、跳舞動畫、犯人逃脫顯示慶祝 GIF。
- ✅ 多人 3–8 人：`buildDeck(N)` 依人數建牌、引擎一般化（計分/輪轉/左右鄰皆隨 N）。
- ✅ **線上連線本體**（房主＝裁判）：Firebase `rooms/{code}/` 下 `public`（公開）/`views/{pid}`（個人手牌+提示，不外洩）/`actions`（玩家動作）；房主 `broadcast()`＋`applyAction()`；客端 `subscribeGame()`。
- ✅ 線上互動：情報交換/交易皆「同時選」；神犬被指定者自選棄；座位相對位置圖（標左鄰=情報交換、右鄰=謠言）。
- ✅ 機器人：房主大廳加/減，名字從 `BOT_NAMES` 隨機取（避開玩家名）；房主端自動代打；機器人不能當房主。
- ✅ 思考時間：逐玩家超時懲罰（30秒，超時自動隨機出牌並降10秒，正常行動回30）；用 Firebase 伺服器時間校準倒數。
- ✅ 房主轉移（房主退出→隨機指派真人）；真人全退（只剩機器人）自動關房。
- ✅ 再來一局：任何人可按回大廳、分數依 pid 累積（機器人也保留，移除機器人其分數消失）；達門檻 `WIN_SCORE` 整場勝→「再玩一場」分數歸0；門檻開房時可設 5–10。
- ✅ 斷線重連：用 localStorage 記住身分 id（`cd_lastpid`）+ 房號，關分頁也能接回原座位（同瀏覽器多分頁測試會搶身分，請用不同瀏覽器/裝置）。
- ✅ 不在場證明被指認可宣告不是犯人、不需棄掉；警部盯著犯人時犯人打出犯人牌→警部當場逮捕(+3)。
- ✅ 🥚 隱藏彩蛋：輪到的人「完全沒牌可打」→放煙火、分數衝到門檻、直接贏整場（被提前換掉手牌則不觸發）。
- ✅ 後門暗號：房主在大廳依序加減機器人 `+2,-1,+3,-2,+4,-3` 後開局→房主拿到彩蛋手牌（偵探,偵探,警部,犯人），且排在先手下一位以確保觸發。
